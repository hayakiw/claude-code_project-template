# /review - コードレビュー

コードの品質、セキュリティ、パフォーマンスをレビューします。

## 使用方法

```
/review [ファイルパスまたはディレクトリ]
```

## レビュー観点

### バックエンド (Python/FastAPI)

1. **コーディング規約**
   - 命名規則（PascalCase/snake_case）
   - docstring（Google形式）
   - 型ヒント

2. **アーキテクチャ**
   - レイヤー分離（router → service → repository）
   - 依存性の注入
   - 例外処理

3. **セキュリティ**
   - SQLインジェクション対策
   - 入力値バリデーション
   - 機密情報の取り扱い
   - 認証・認可の実装

4. **パフォーマンス**
   - N+1問題
   - 非同期処理の適切な使用
   - 不要なDB呼び出し

### フロントエンド (Next.js/TypeScript)

1. **コーディング規約**
   - コンポーネント命名
   - Props型定義
   - Tailwind CSSの使用

2. **React/Next.js**
   - "use client" の適切な使用
   - useEffect の依存配列
   - 状態管理

3. **セキュリティ**
   - XSS対策
   - 機密情報のハードコード
   - 適切なエラーハンドリング

4. **パフォーマンス**
   - 不要な再レンダリング
   - 画像最適化
   - バンドルサイズ

## 出力フォーマット

```markdown
## レビュー結果: [ファイル名]

### 問題点

#### [重要度: 高/中/低] 問題タイトル
- **場所**: ファイル:行番号
- **問題**: 問題の説明
- **修正案**:
\`\`\`python
修正後のコード
\`\`\`

### 改善提案

- 改善点1
- 改善点2

### 良い点

- 良い点1
- 良い点2
```

## レビューチェックリスト

### 必須チェック項目

- [ ] エラーメッセージは日本語か
- [ ] バリデーションは適切か
- [ ] 例外処理は適切か
- [ ] ログ出力は適切か
- [ ] セキュリティ上の問題はないか

### 推奨チェック項目

- [ ] テストは書かれているか
- [ ] ドキュメントは更新されているか
- [ ] パフォーマンス上の問題はないか
- [ ] コードの重複はないか
